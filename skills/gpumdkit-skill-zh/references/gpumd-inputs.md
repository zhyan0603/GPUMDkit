# GPUMD 输入文件

在创建或审核 `model.xyz`、`run.in`、`kpoints.in` 或模态分析输入时，请使用此参考。

## 目录

- 参考范围
- `model.xyz`
- 分组
- `run.in`
- 辅助文件
- 输入审核

## 参考范围

本文件包含正常 GPUMD 设置所需的可移植输入文件规范。根据任务使用的关键字，将其与设置、系综、计算和输出目录配对使用。

## `model.xyz`

GPUMD 读取扩展 XYZ 格式。

第 1 行：

```text
<number_of_atoms>
```

第 2 行包含大小写不敏感的 `keyword=value` 对。GPUMD 识别以下字段：

| 头字段 | 要求 | 含义 |
|---|---|---|
| `lattice="ax ay az bx by bz cx cy cz"` | 必需 | 以 Angstrom 为单位的盒子矢量 |
| `pbc="T T T"` | 可选；默认 `T T T` | 沿盒子矢量的周期性 |
| `properties=...` | 实际必需 | 每原子列的模式 |

识别的属性条目：

| 属性 | 要求 | 单位/含义 |
|---|---|---|
| `species:S:1` | 必需 | 大小写敏感的化学符号 |
| `pos:R:3` | 必需 | 笛卡尔坐标，Angstrom |
| `mass:R:1` | 可选 | amu；缺失时使用 GPUMD 默认质量 |
| `charge:R:1` | 可选 | 基本电荷；默认为零 |
| `vel:R:3` | 可选 | Angstrom/fs |
| `group:I:N` | 可选 | `N` 个整数分组方法 |

每个原子行必须恰好包含 `properties` 声明的列数。原子行数必须等于第 1 行。

包含一个分组方法的示例：

```text
2
lattice="5 0 0 0 5 0 0 0 5" pbc="T T T" properties=species:S:1:pos:R:3:group:I:1
Li 0.0 0.0 0.0 0
O  2.5 2.5 2.5 1
```

不要将此盒子、元素或分组复制到实际计算中。根据用户的结构和预期命令来构建它们。

## 分组

`group:I:N` 在每个原子行末尾附加 `N` 个整数标签。分组方法索引从 `0` 到 `N-1`。分组感知命令通常需要 `<grouping_method>` 和 `<group_id>` 两个参数；一些遗留形式默认使用分组方法 0。

分组标签不定义化学元素。仅在命令或下游工具需要子集、区域、分子或按元素分组时才添加它们。验证每个引用的分组包含预期的原子。

## `run.in`

每个有效行的格式为：

```text
keyword parameter_1 parameter_2 ...
```

空行和 `#` 之后的内容会被忽略。命令按文件顺序执行。典型的协议如下：

```text
replicate ...          # 可选，在需要时放在势函数之前
potential ...
minimize ...           # 可选的即时操作
velocity ...
time_step ...

ensemble ...
dump_thermo ...
compute_... ...
dump_... ...
run ...

ensemble ...           # 下一阶段
compute_... ...         # 重新发出非传播命令
run ...
```

不要将此作为现成的协议使用。加载相关参数参考并询问阶段特定的值。

命令生命周期很重要：

- 即时操作如 `potential`、`velocity`、`minimize` 和一些静态计算在解析时立即执行。
- `time_step` 在后续 `run` 块中传播。
- 系综、大多数计算、dump、变形、约束和驱动场是运行作用域的或非传播的。
- `run <number_of_steps>` 执行当前阶段且不传播。

除非附带条目明确描述了即时或传播操作，否则将不熟悉的命令视为运行作用域。如果解析器行为不同，请停止并确认可执行文件版本，而不是猜测。

## 辅助文件

| 文件 | 使用者 | 内容 |
|---|---|---|
| `kpoints.in` | `compute_phonon` | 分数高对称性 k 点和标签；空行分隔路径 |
| `eigenvector.in` | GKMA/HNEMA/模态分析 | 由文档化的模态工作流生成的谐波特征向量 |
| 势函数文件 | `potential` | 特定格式的模型参数；保留提供的文件并在使用前验证其头/元素信息 |
| `zbl.in` | 灵活 NEP ZBL | 模型使用的按对排序的 ZBL 参数 |
| PLUMED 输入 | `plumed` | 匹配编译特性集的 PLUMED 配置 |

## 输入审核

- 确认晶格行列式、盒子方向、PBC 和坐标单位。
- 确认元素和顺序与势函数文件匹配。
- 确认每个分组方法/ID 和电荷/质量/速度列。
- 确认没有原子行缺失或有多余字段。
- 确认 `run.in` 中的每个关键字和签名存在于附带的参数目录中。
- 确认所有文件名从执行目录中解析。
- 确认早期运行的追加模式输出不会污染本次运行。
