# NEP 训练路由与工作流

每次执行 NEP 训练、微调、预测或部署任务时，请先阅读本文件。然后根据任务需要加载数据、参数和输出参考文件。

## 目录

- 内置参考文件范围
- 任务路由
- 选择运行模式
- 训练工作流
- 预测与 GPUMD 部署
- 版本敏感事项

## 内置参考文件范围

本技能中的 NEP 文件是一套自成体系、分类整理的参考摘要，涵盖输入数据格式、所有当前 `nep.in` 关键字、默认值、约束条件、输出文件和部署检查。它们不依赖 GPUMD 源代码树。

以解析器消息和可执行文件版本输出作为运行时证据。如果本地行为与参考不同，请停止操作并报告具体的行号、错误和版本信息，然后向用户确认版本相关的具体行为。不要凭记忆替换语法。

## 任务路由

| 需求 | 参考文件 |
|---|---|
| `train.xyz`/`test.xyz` 字段、单位、模型类型目标、验证 | `references/nep-data.md` |
| 所有当前 `nep.in` 关键字、语法、约束和默认值 | `references/nep-parameters.md` |
| `loss.out`、模型/重启文件、拟合图输出、预测输出 | `references/nep-outputs.md` |
| 转换、过滤、采样、绘图 | 对应的 GPUMDkit 模块参考文件 |
| 在 MD 中使用 `nep.txt` | `references/gpumd.md` 和 GPUMD 参数参考文件 |

## 选择运行模式

请用户选择并定义验收标准：

| 模式 | 关键设置/文件 |
|---|---|
| 势函数训练 | `model_type 0`, `prediction 0`, `type`, `train.xyz`, 可选 `test.xyz` |
| 偶极矩训练 | `model_type 1`, 偶极矩目标, 全局/原子级选择 |
| 极化率训练 | `model_type 2`, 极化率目标, 全局/原子级选择 |
| qNEP 势函数 | `model_type 0`, `charge_mode 1` 或 `2`, 电荷相关损失/设置 |
| 预测 | `prediction 1`, `nep.txt`, `train.xyz` 中的结构 |
| 基础模型微调 | `fine_tune <model> <restart>`, 兼容的固定架构, 新的 `type` |
| 恢复中断的训练 | 兼容的 `nep.in` 加工作目录中的 `nep.restart` |

不要从数据集文件名推断模型类型。确认预期的下游可观测量和标签。

## 训练工作流

1. 保留原始参考数据，记录 DFT 代码/设置、单位、赝势、泛函和数据来源。
2. 加载 `nep-data.md`；转换为经过验证的 extxyz 格式，审核组成、晶胞、标签、范围、最小距离、重复结构以及训练/测试集泄漏。
3. 确认模型类型、元素种类数/顺序、数据集划分、权重和缺失目标。
4. 加载 `nep-parameters.md`；与用户确认所有非平凡的架构、损失、优化器、检查点、电荷和 ZBL 决策。
5. 按需写入 `nep.in`、`train.xyz` 和 `test.xyz`。记录每个值是用户选择的、继承的还是文档默认值。
6. 预检工作目录文件、可执行文件路径/构建、GPU/运行时、预期持续时间和输出冲突行为。
7. 仅在获得明确授权后运行。记录命令、工作目录、源代码版本/构建、环境、退出状态和警告。
8. 加载 `nep-outputs.md`；监控训练/测试损失，按属性、元素种类、组成和构型类别检查拟合图输出。
9. 在用户目标工况下评估稳定性和外推能力，然后才批准模型用于正式 MD 模拟。

不要仅凭训练 RMSE 就宣布成功。精度阈值、数据集大小、代数和停止策略因体系和应用而异。

## 预测与 GPUMD 部署

预测模式仅评估 `train.xyz` 中的结构，且需要 `nep.txt`。使用干净的预测目录以避免混淆训练和预测输出。

在 GPUMD 中使用模型之前：

- 验证元素种类和类型顺序是否与 `model.xyz` 一致；
- 验证势函数格式/版本和电荷/k 空间需求；
- 确认训练域覆盖了待采样的组成、相态、温度、应变、缺陷、表面和短程距离；
- 定义用户批准的外推/稳定性检查；
- 在正式采样阶段前进行授权的简短冒烟测试。

GPUMD 命令通常为：

```text
potential <path-to-nep.txt>
```

使用前确认模型头部、元素种类顺序以及任何电荷/k 空间需求。

## 版本敏感事项

- 本内置版本快照仅接受 `version 4`。如果本地解析器行为不同，请报告可执行文件版本并等待版本确认。
- 当前 `l_max` 是三体限制加可选的 0/1 描述符开关，而非旧的三数 NEP3 形式。
- 当前微调语法为 `fine_tune <nep_model_file> <nep_restart_file>`，而非 `fine_tune 1`。
- `nep.restart`（带点号）是当前文档记录的重启文件名；旧文本可能使用 `nep_restart`。
- 不要从旧技能复制超参数默认值。使用 `nep-parameters.md`，并区分默认值和用户确认的正式训练参数。
