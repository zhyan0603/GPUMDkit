# NEP 训练与预测数据

使用本参考文件创建和审核 `train.xyz` 和 `test.xyz`。

## 目录

- 参考范围
- 构型格式
- 各模型类型的目标
- 单位与约定
- 数据集审核与划分

## 参考范围

本文件包含可移植的 `train.xyz` 和 `test.xyz` 规范。两个文件均使用多帧扩展 XYZ 格式；每个构型占 `N + 2` 行。

## 构型格式

第 1 行：

```text
<number_of_atoms>
```

第 2 行包含头部字段。每个构型都必须包含 `lattice`、`energy` 和 `properties`，即使后续非势函数模型会忽略能量和力相关目标：

| 字段 | 含义 |
|---|---|
| `lattice="ax ay az bx by bz cx cy cz"` | 必需的晶胞矢量，单位为埃（Angstrom） |
| `energy=<total_energy>` | 必需的总能量字段，单位为 eV；偶极矩/极化率训练时忽略 |
| `virial="9 components"` | 可选的全晶胞维里张量，单位为 eV |
| `stress="9 components"` | 可选的应力张量，单位为 eV/Angstrom^3；如果存在 virial 则忽略 |
| `weight=<relative_weight>` | 可选的构型权重 |
| `properties=...` | 必需的逐原子列格式 |

相关逐原子属性：

```text
species:S:1
pos:R:3
force:R:3        # 或 forces:R:3
bec:R:9          # 可选的 Born 有效电荷目标
```

仅为示例格式：

```text
properties=species:S:1:pos:R:3:force:R:3
```

不要复用示例中的能量、晶胞或结构。确认每行原子数据与声明的格式和第 1 行的原子数一致。

## 各模型类型的目标

| 模型 | 必需/使用的目标 |
|---|---|
| 势函数（`model_type 0`） | 能量和力；当预期损失中包含时还包括维里/应力 |
| 偶极矩（`model_type 1`） | 头部 `dipole="dx dy dz"`；忽略能量/维里/应力/力 |
| 极化率（`model_type 2`） | 头部 `pol="9 components"`；忽略能量/维里/应力/力/偶极矩 |
| qNEP 势函数 | 势函数目标；不需要目标电荷；可选 `bec:R:9` 用于 BEC 拟合 |

对于偶极矩和极化率，确认目标是全局还是逐原子的，并一致地设置 `atomic_v`。不要混用不兼容的目标约定。

## 单位与约定

| 物理量 | 单位/约定 |
|---|---|
| 长度/位置 | Angstrom |
| 能量 | eV，输入中为全晶胞总量 |
| 力 | eV/Angstrom |
| 维里 | eV，输入中为全晶胞总量 |
| 应力 | 输入中为 eV/Angstrom^3 |
| BEC | 基本电荷 |
| 偶极矩/极化率 | 用户选择的一致单位；请记录 |

维里输入中正值表示压缩，负值表示拉伸。应力输入使用相反的符号约定：正值表示拉伸，负值表示压缩。如果两者同时存在，使用维里。

NEP 训练假设所有方向均为周期性边界条件。对于平板、团簇或真空体系，请向用户确认周期性晶胞和参考计算如何表征该体系。

对于厚度小于径向截断距离两倍的晶胞，代码可能会在内部进行复制。请验证这在物理上是否合理。

训练使用单精度。低于 `-100 eV/atom` 的参考能量可能会损失精度。报告此限制并询问如何处理参考能量；不要静默地进行平移。

## 数据集审核与划分

- 验证元素符号和精确的 `type` 顺序。
- 验证能量是全晶胞总量而非每原子能量；验证维里/应力的符号和单位。
- 确认每帧的力列和原子顺序。
- 检测 NaN/Inf、格式错误的晶胞、极端距离、重复构型和不一致的标签。
- 按组成和构型类别检查能量/力/维里范围。
- 当独立性很重要时，防止轨迹相邻构型在训练/测试集之间泄漏。
- 保留原始数据，记录过滤条件、权重、排除项和随机种子。
- 确保根据用户模型范围，涵盖了所有预期的元素种类、相态、组成、应变、缺陷、表面、温度和短程工况。
- 不要自行设定训练/测试比例或最小数据集大小。

在确认转换标准后，使用 `references/format-conversion.md`、`analyzers.md` 和 `sampling.md` 中的 GPUMDkit 命令。
