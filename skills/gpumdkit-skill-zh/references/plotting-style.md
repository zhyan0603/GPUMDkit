# GPUMDkit 绘图风格

新建绘图脚本、调整现有图的外观或审查绘图贡献时，读取本参考。同时读取 `visualization.md`、`contributing.md` 以及所绘物理量对应的参考。本指南从 `Scripts/plt_scripts` 的重复设计中蒸馏而来，不覆盖各脚本的特定行为。

## 设计特征

GPUMDkit 图像是紧凑、面向论文的科学图。整体使用白色背景、克制的 Matplotlib 配色、带单位的直接物理量标签，并提供足够但不过度的注释。数据及其不确定性应保持视觉主导。

项目中有两类既有风格，两者都有效：

- 通用分析和 NEP 绘图多使用 Matplotlib 面向对象 API、无衬线字体栈、紧凑多面板布局和 `tight_layout()`。
- 热输运和声子绘图保留较早的 `pylab` 风格、四边向内刻度、次刻度、更粗的最终/平均曲线以及无边框图例。

不要强制把现有脚本从一类迁移到另一类。应匹配同一科学工作流中的相邻脚本。

## 视觉语言

### 字体

- 优先使用 `sans-serif`，字体回退顺序为 `Arial`、`DejaVu Sans`、`Liberation Sans`。
- 坐标标签使用数学文本并明确单位，例如 `Å$^2$/ps`、`eV/atom` 和 `W/mK`。
- 同一图中的标签和刻度字号保持一致。既有图通常约为 10-13 pt，但这是观察结果，不是全局强制值。
- 图例和注释保持简洁。拟合统计量或派生数值应靠近对应面板或曲线。

### 颜色与线条层级

- 相关分量使用 Matplotlib 循环色（`C0`、`C1`、`C2` 等）。当目标脚本已经采用该约定时，方向数据通常将 `x`、`y`、`z` 映射为 `C0`、`C1`、`C2`。
- 参考线、原始重复轨迹和恒等/零线使用灰色或黑色。对比图通常使用灰色虚线作为恒等参考。
- 噪声较大的重复数据或分布使用较低透明度；平均、累积结果、拟合或最终轨迹使用更粗的不透明线条突出。
- 不确定性使用半透明带；需要与拟合线区分的拟合点可使用空心标记。
- 保留脚本现有的颜色语义。不得通过换色改变相关图中各分量的含义。

### 坐标轴与布局

- 根据科学比较关系选择布局：紧凑单面板、对齐的 1xN 对比或 2x2 训练摘要都是既有模式。
- 使用 `tight_layout()`；若其无法保持目标面板几何，则使用明确的 `subplots_adjust(...)`。
- 保持坐标轴简洁。通常不显示网格；输运图的图例常无边框。
- 对比面板使用相同数据范围并绘制对角参考线。损失或跨多个数量级的数据适合使用对数坐标。
- 仅当不同尺度或诊断信息确实改善解释时使用嵌入坐标轴，例如残差分布或早期时间行为。

### 保存

- 保留各脚本现有且灵活的 `save` 参数位置，不建立全局参数契约。
- 保留每个脚本自己的显示 DPI 和保存 DPI。保存图常为 300 DPI PNG，有时使用 `bbox_inches='tight'`，但项目明确拒绝统一 DPI。
- 现有命令应保持输出文件名及大小写。新命令使用与图类型相关、稳定且有描述性的文件名。
- 若相邻脚本默认交互显示，则保持该行为；通过路由命令显式选择保存。

## 科学绘图规则

- 不得编造或静默改变拟合窗口、平滑窗口、截断值、排除规则、分量映射或单位换算。
- 用视觉编码和标签区分原始数据、变换数据、拟合、平均值与不确定性。
- 每个有量纲的坐标轴都必须标单位；派生量也应带单位报告。
- 对比图应绘制恒等线并报告脚本实际计算的误差指标。不能只凭视觉吻合暗示验证质量。
- 收敛与输运图中，若各次运行可表达不确定性，应保留其可见性，再突出聚合结果。
- 绘图前验证数组形状、有限值和预期列。输入缺失或格式错误时输出面向用户的错误并停止，不要生成误导性的空图。

## 新绘图工作流

1. 找到同一科学类别中最接近的脚本，复用其图形语法与 CLI/保存行为。
2. 依据项目证据或用户决定确认输入、列、单位、分量映射、拟合/平滑选择和输出名。
3. 遵循 `contributing.md` 中的文件头、帮助、错误信息、依赖和导入顺序规则。
4. 在有助于可读性时，将数据读取与验证和绘图辅助函数分离；为辅助函数添加 docstring。
5. 使用能够表达结果的最小布局，并保留原始数据语境。
6. 用代表性数据测试交互与保存路径；检查标签、单位、图例、裁切、空数据、NaN/Inf 和无界面保存。
7. 运行 `contributing.md` 的验证清单；用户可见的新功能必须同步更新中英文文档。

## 审查清单

- 绘图匹配相邻 GPUMDkit 工作流，而非引入无关的全新“家族风格”。
- 颜色、标记、线条与透明度都对应明确记录的数据差异。
- 标签包含物理量和单位；图例简洁且不遮挡数据。
- 恒等线、零线、拟合和不确定性与观测数据可清楚区分。
- 科学选择来自用户输入、文档默认或已确认的项目设置。
- 新脚本的 `-h` 与缺参行为在没有可选/重量级依赖时也可工作。
- 显示和保存路径均可工作；文件名与 DPI 遵循目标脚本，不做全局重写。
- 不得以外观清理为名改变绘图逻辑。

## 实际依赖

绘图目录不存在一套统一依赖。Matplotlib 和 NumPy 最常见；单个脚本还可能需要 pandas、SciPy、seaborn、ASE、scikit-learn、UMAP、calorine 或 `ferrodispcalc`。执行前检查目标脚本；新脚本应将可选依赖导入放在帮助处理之后。
