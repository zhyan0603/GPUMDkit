## 功能简介

GPUMDkit 为计算材料科学中的常见任务提供统一入口——格式转换、结构分析、性质计算和可视化——无需编写自定义脚本。

## 准备工作

### 1. 使用 Conda 安装（推荐）

```bash
conda create -n gpumdkit -c gpumdkit -c conda-forge gpumdkit
conda activate gpumdkit
```

该方式会安装 GPUMDkit 及其标准依赖，无需克隆仓库或配置环境变量。

部分功能需要额外安装可选依赖：

```bash
pip install neptrain calorine
```

### 2. 从源码安装（可选）

如需使用源码，请克隆仓库并运行安装脚本：

```bash
git clone https://github.com/zhyan0603/GPUMDkit.git
cd GPUMDkit
source ./install.sh
```

安装脚本会写入 `GPUMDkit_path` 和 `PATH` 设置，并在当前 shell 中加载。默认写入 `~/.bashrc`；如果当前 shell 是 zsh，则写入 `~/.zshrc`。如果检测到已有 GPUMDkit 路径，安装脚本会打印旧路径并询问是否替换；修改 rc 文件前会先创建备份。

典型安装输出：

```text

======================================================
```

### 3. 验证安装

```bash
gpumdkit.sh -h
```

此命令会打印帮助表，列出所有可用选项。

更实用的首次检查是运行：

```bash
gpumdkit.sh -doctor
```

终端会分别显示 **Environment**、**Common packages** 和 **Optional packages**。
可选包显示 `MISS` 并不表示安装失败：只有需要对应功能时才安装。例如，DeepMD
转换需要 `dpdata`，NEP 辅助计算会使用 `calorine`。这样可以在开始流程前先定位缺少的模块。

## 运行前先读懂命令

CLI 示例中使用以下记号：

| 记号 | 含义 | 示例 |
|---|---|---|
| `<必填项>` | 替换为必须提供的实际值；不要输入尖括号。 | `<output.xyz>` → `model.xyz` |
| `[可选项]` | 可以省略；脚本将采用文档说明的行为。 | `[max_corr_steps]` |
| `...` | 后面可以跟随一个或多个值。 | `<element...>` |

对于不熟悉的 Python 命令，先让命令本身解释参数。例如：

```text
$ gpumdkit.sh -calc msd -h
 Usage: gpumdkit.sh -calc msd <extxyz_file> <element_symbol> <dt_fs> [max_corr_steps]

 Arguments:
   extxyz_file       Path to the input extxyz trajectory file
   element_symbol    Chemical symbol (e.g., Li, O, Na)
   dt_fs             Time step between consecutive frames (fs)
   max_corr_steps    Max correlation lag steps (optional)
```

这里的 `dt_fs` 是**相邻已写出轨迹帧**之间的时间间隔（fs），不会从 MD 输入文件中自动推断；应以自己的轨迹写出设置为准。绘图模块请用 `gpumdkit.sh -plt -h` 查看图类型；各图参数的位置见[绘图脚本](绘图脚本.md)。

交互菜单中的 `------------>>` 表示等待输入。许多 CLI 命令结束后还会打印 `Code path:`，可据此定位产生结果或报错的实现脚本。

## 命令行模式

直接命令使用固定的位置参数：

```bash
gpumdkit.sh -<选项> [参数...]
```

示例：

```bash
gpumdkit.sh -pos2exyz POSCAR model.xyz
gpumdkit.sh -plt train
gpumdkit.sh -calc msd trajectory.xyz Li 10
```

第一个示例读取 `POSCAR`，写出 `model.xyz`。

## Hello World 示例

一个最小化的端到端检查，确认安装是否正常——创建一个极小的硅 POSCAR，转换为 extxyz，并查看输出：

```bash
cat > POSCAR << 'EOF'
Si
1.0
0 2.715 2.715
2.715 0 2.715
2.715 2.715 0
Si
1
direct
0 0 0
EOF
gpumdkit.sh -pos2exyz POSCAR model.xyz
head model.xyz
```

如果 `model.xyz` 显示一个 Si 原子及晶格信息，说明安装成功。

> **什么是 extxyz？** extxyz 是扩展 XYZ 格式：第 1 行是原子数，第 2 行包含晶格及结构级属性（按需包含能量/力/维里），第 3 行及以后是每个原子及其原子级属性。它是 NEP 的原生训练数据格式。

## 常见示例

### POSCAR 转 extxyz

```bash
gpumdkit.sh -pos2exyz POSCAR model.xyz
```

### 添加 GPUMD group 标签

```bash
gpumdkit.sh -addgroup POSCAR Li Y Cl
```

group 标签用于一些需要原子分组的 GPUMD 相关流程和分析，例如特定元素的 MSD 或扩散性质计算。
