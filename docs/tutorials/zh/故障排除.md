## 常见问题

### 1. 安装后在新终端中提示 `gpumdkit.sh: command not found`

**症状：** 打开新终端输入 `gpumdkit.sh` 时提示 `command not found`，尽管安装过程看似成功。

**诊断：**

```bash
echo $GPUMDkit_path
```

应打印出安装目录。如果为空或不正确，说明 shell 未加载环境变量。

**修复：**

```bash
source ~/.bashrc        # Linux / bash
source ~/.zshrc         # macOS / zsh
```

如果 source 后变量仍为空，请在 GPUMDkit 目录下重新运行安装脚本：

```bash
cd /path/to/GPUMDkit
bash install.sh
```

---

### 2. 绘图时报错 `FileNotFoundError: loss.out`（或 `thermo.out`、`msd.out` 等）

**症状：** 运行 `gpumdkit.sh -plt <type>` 时因找不到某个 `.out` 文件而报 `FileNotFoundError`。

**诊断：**

```bash
ls *.out
```

如果没有输出，说明当前目录不包含 GPUMD/NEP 的输出文件。

**修复：**

```bash
cd /path/to/your/simulation/run
gpumdkit.sh -plt <type>
```

务必在包含输出文件的目录中运行 `-plt`，或传入正确的路径。

---

### 3. 运行某些计算器时报 `No module named 'calorine'`（或 `'neptrain'`、`'dpdata'`）

**症状：** 计算器或转换器中止并报 `ModuleNotFoundError: No module named 'calorine'`（或 `neptrain`、`dpdata`）。

**诊断：**

```bash
pip list | grep -iE 'calorine|neptrain|dpdata'
```

如果缺少该包，请安装。

**修复：**

```bash
pip install calorine neptrain dpdata
```

如果使用 conda，请在已激活的 `gpumdkit` 环境中运行对应的 `conda install` 或 `pip install`。运行 GPUMDkit 前请确保环境已激活。

---

### 4. MSD 曲线剧烈震荡而非线性扩散

**症状：** MSD 曲线噪声很大或来回震荡，没有随时间近似线性增长，导致扩散系数不可靠。

**诊断：**

检查模拟中实际使用的帧间隔：

```bash
head thermo.out
```

将那里的采样间隔与你传给 `-calc msd` 的 `dt_fs` 进行对比。

**修复：**

- 确保传给 `gpumdkit.sh -calc msd` 的 `dt_fs` 等于真实 MD 时间步长乘以采样（输出）间隔，而不仅仅是 MD 时间步长。
- 确认轨迹长度足以覆盖你体系的扩散时间尺度。
- 注意：低温下固体可能确实不会出现线性 MSD 区域（原子被局域化），这是物理现象，不是 bug。

---

### 5. `-out2xyz` 或 `-pos2exyz` 提示 "No files found"

**症状：** 格式转换命令报告找不到输入文件，尽管你预期文件就在那里。

**诊断：**

```bash
ls *.vasp *.xyz *.OUTCAR
```

如果没有匹配项，说明你所在目录不对，或文件扩展名与命令预期的不一致。

**修复：**

- 运行转换前先 `cd` 到实际包含结构文件的目录。
- 确认文件扩展名与预期一致（例如 `POSCAR`、`*.vasp`、`OUTCAR`、`*.xyz`）。
