    GPUMDkit 是什么？

    • 面向 GPUMD 与 NEP 的一站式命令行工具箱。
    • 将常见流程标准化：数据准备、采样筛选、训练分析、批处理工作流。
    • 支持交互菜单与参数化命令，适配新手和自动化场景。

    核心价值

    降低重复脚本维护成本，统一数据处理链路，提升训练与分析效率。

    覆盖模块

    format_conversion / sample_structures / workflow / calculators / analyzer / plt_scripts。

    安装与环境建议

    git clone https://github.com/zhyan0603/GPUMDkit.git
cd GPUMDkit
source ./install.sh
source ~/.bashrc

    推荐环境

    独立 Conda/Python 环境，安装 neptrain、ase、pymatgen、dpdata 等依赖。

    启动方式

    gpumdkit.sh（交互）
    gpumdkit.sh -h（命令行）

    交互模式 vs 命令行模式

    交互模式（新手友好）

    • 菜单引导清晰
    • 降低参数记忆成本
    • 适合探索功能与流程学习

    命令行模式（自动化友好）

    • 直接执行，速度快
    • 适合集群脚本与批处理
    • 可融入自定义 pipeline
    gpumdkit.sh -out2xyz .
gpumdkit.sh -calc ionic-cond Li 1
gpumdkit.sh -plt train

    模块总览 I：数据准备与组织

    Format Conversion

    VASP/CP2K/ABACUS/LAMMPS/CIF 与 extxyz/POSCAR 等互转。

    Sample Structures

    uniform/random/FPS、扰动结构、最大力偏差筛选。

    SCF 与 MD 批处理预处理，组织目录与提交脚本。

    离子电导、DOAS、NEB、描述符、极化分析等。

    能量/力范围、最短距离、过滤、组分分析、离群点排查。

    Plot Scripts

    活性学习工作流（workflow_active_learning）

    1. 基于当前 NEP 模型进行 MD 采样与候选结构生成。
    2. 过滤非物理结构（最短距离/盒子限制等规则）。
    3. 按策略抽样（uniform/random/pynep/neptrain）控制 FP 数量。
    4. 提交 SCF 计算、回收标注数据并并入训练集。
    5. 重新训练并评估 NEP，进入下一轮迭代。
    适用于“采样—标注—训练—评估”闭环加速。

    极化材料分析（Polar Material Analysis）

    核心命令

    -calc nlist / disp / avg-struct / oct-tilt / pol-abo3
    -plt plane-grid

    适用场景

    ABO3 体系局域位移、八面体倾斜、极化向量统计与层内矢量可视化。

    实用命令速查

    # 数据转换
gpumdkit.sh -out2xyz .
gpumdkit.sh -pos2exyz POSCAR model.xyz
gpumdkit.sh -lmp2exyz dump.lammpstrj Li Y Cl
    # 数据分析
gpumdkit.sh -range train.xyz force
gpumdkit.sh -min_dist_pbc train.xyz
gpumdkit.sh -analyze_comp train.xyz
    # 计算与筛选
gpumdkit.sh -calc ionic-cond Li 1
gpumdkit.sh -calc des train.xyz des.npy nep.txt Li
gpumdkit.sh -pynep
    # 结果可视化
gpumdkit.sh -plt train
gpumdkit.sh -plt thermo
gpumdkit.sh -plt hnemd

    扩展能力：Custom Commands

    通过配置 ~/.gpumdkit.in 可以注册个人命令别名并组合常用流程。

    custom_quick_check() {
  gpumdkit.sh -range "$1" force
  gpumdkit.sh -min_dist_pbc "$1"
  gpumdkit.sh -analyze_comp "$1"
}
# 用法：gpumdkit.sh -quick_check train.xyz
    建议将高频流程沉淀为自定义命令，提高复用效率。
